plainfame CMS

Das webCMS plainfame entstand anno 2012 quasi als Nebenprodukt meiner damaligen Arbeit, um ein leichtgewichtiges, schnell anpassbares CMS zur Hand zu haben.
Natürlich gibt es für die verschiedenen features unterschiedliche Herangehensweisen. Ziel bei der Entwicklung ist es hier, einen Tradeoff zwischen einfachster und bester Lösung zu finden.

Folgende Konzepte wurden umgesetzt:
  • lightweight bedeutet wenig Balast
    • keine Datenbank nötig
    • das CMS funktioniert ohne Cookies (und damit ohne Session)
    • keine Klassenhierarchie, sondern one-purpose prozeduraler Code
  • Keep it simple: es gibt keine "globale" Variablen: Jedes Modul hat seinen eigenen scope
  • leicht zu konfigurieren

Das CMS ist wie erwähnt modular aufgebaut. Jede Seite kann entweder reines HTML oder PHP-Code sein
Als Appetithappen sind folgende Module gleich an Bord:
  • Verwaltung von Unterseiten mit Tabs (jQuery)
  • Kontaktformular (HTML5, PHP)
  • Rating Modul, mit dem z.B. über Bilder oder Meinungen abgestimmt werden kann (jQuery, PHP)
  • Geschachteltes Karusell für Bilder (jQuery)
  • Laden von Tabellen aus CSV-Dateien (PHP)

Weitere features

  1. Oberes Menü fixiert sich beim Scrollen (funktioniert ohne Javascript)
  2. Eingebettete Schriftart Open Sans
  3. Modularer Aufbau, jede Seite kann entweder reines HTML oder PHP-Code sein
  4. Für die Auswahl der jeweiligen Sprache wird die Sprachpräferenz im Browser verwendet
  5. Inhaltsdateien OHNE Länderkennung wird als Fallback genutzt, falls eine Inhaltsdatei mit Länderkennung angefragt wird, diese aber nicht existiert

Installation/Konfiguration

  1. Alle Dateien in den htdocs Ordner ablegen
  2. Unterseiten in pagedata.php eintragen (unter Umständen zweisprachig)
  3. Impressum anpassen
  4. Für Kontakt: Emailadressen anpassen
  5. Für Votes: Der data-Ordner muss für PHP Schreibrechte haben
